## Spec details

### #1 — size_tier thresholds (calibrated against current top 25)

```python
def classify_size_tier(years: float, reviews: int) -> str:
    if years >= 20 and reviews >= 400: return "XL"
    if years >= 15 or reviews >= 300:  return "L"
    if years >= 10 or reviews >= 150:  return "M"
    return "S"
```

Expected distribution on current top 25: ~10 L / 9 M / 6 S / 0 XL. XL is a reserve tier for larger contractors outside the hidden-gems pool — no top-25 contractor meets it today, which is fine.

### #2 — FSM-vendor disqualifier

Brand names to match against cached SerpAPI job descriptions (case-insensitive, word-boundary):

```
ServiceTitan, Service Titan, Housecall Pro, HousecallPro, Jobber,
FieldEdge, Field Edge, ServiceBridge, Service Bridge, Workiz,
FieldPulse, Field Pulse, Service Fusion, ServiceFusion, Tradify,
Kickserv, mHelpDesk, Synchroteam, GorillaDesk, WorkWave
```

When a contractor's job descriptions contain ANY of these brand names as a required skill or tool, apply a **−15 point disqualifier** labeled `already_fsm_customer` and suppress them from the top 25 entirely. This is a harder disqualifier than `smooth_ops` because it's explicit proof of vendor relationship, not inferred from review language.

Add an audit sidecar: `data/03_hidden_gems/already_fsm_dropped.csv` so we can see who got kicked and verify the regex isn't overreaching.

### #3 — ICP fit dimension

```python
def score_icp_fit(r: pd.Series) -> tuple[float, list[str]]:
    cls = safe_str(r.get("class"))
    if cls == "CR-39":
        return 5.0, ["Dual-scope license (commercial + residential) (+5)"]
    if cls in {"R-39", "R-39R"}:
        return 2.0, ["Residential-only license (+2)"]
    return 0.0, []
```

New row in scored.csv: `score_icp_fit`. Added to `score_total`.

### #4 — signal_freshness display field

Compute at render time in `14_dossier_cards.py`:

- Pull all dated signal mentions from LLM cache (pain_mentions, momentum_mentions, switcher_mentions, dispatch extractions) — each has a date via review_index
- Count how many fall in the last 30 days vs 31-90 days vs 91-180 days
- Compute `freshness_ratio = last_30_days / total`
- Render a small badge on the header:
  - `freshness_ratio >= 0.5`: "FRESH — 50%+ of signals in last 30 days"
  - `freshness_ratio >= 0.25`: "RECENT — 25%+ of signals in last 30 days"
  - else: no badge (it's already within the 180-day window, no need for a stale marker)

Non-scoring, display-only. Rep uses it to sort "call these first."

### #5 — Document Apollo coverage gap

Add a paragraph to `SCORING.md` under the "What the score does NOT try to do" section, along the lines of:

> **Firmographic enrichment is sparse for hidden-gem contractors.** We check Apollo's organization database for employee count and estimated revenue, but Apollo's coverage of sub-$10M private HVAC shops is near zero — 0 of the 25 contractors in the current top list have Apollo firmographics. This is expected and arguably a feature: the pipeline is surfacing prospects that commodity lead databases miss. For size estimation we rely on our own `size_tier` computed from license tenure and review volume.
